面向对象

学习目标

一、对象

javascript 对象相关概念 ：1.构造函数 2. 实例对象 3.原型

1.1 什么是对象

抽象—–：构造函数； 具体—-实例对象

  1. 属性
  2. 行为 新时代的女性都会coding

1.2 .属性 和 [属性] 的区别

  • 对象成员（属性和方法）的访问方式：
  1. 对象名.属性
  2. 对象名[‘属性名称’]
  • 区别：方括号的方式可以使用变量

1.3 对象创建的方式

  1. 构造函数：new

  1. 字面量

  2. 三种方式的等效创建

  3. 构造函数实例化本质上做了什么工作？

1. 开辟了堆内存用来存储实例中数据（属性和方法）
2. 用this指向该区域
3. 通过this向该区域中放置数据
4. 返回this
  1. 构造函数的返回值问题
    1. 构造函数默认的返回值就是this
    2. 如果返回的是基本数据类型，那么和不返回等效
    3. 如果返回的是引用数据类型，那么就以此为准，就不返回this了

1.4 从程序的角度解析对象

  1. 对象的本质：无序的键值对集合

1.5 原型

减少内存消耗，实现数据共享 ；js的专属，别的语言不存在

  • 函数都有一个原型属性：prototype,该属性的值本质上也是对象（实际就是object的实例）
  • 作用：实现数据共享（实例对象之间进行共享） 实现继承

  • prototype方法，在实例对象中创建了一个

  • 通过对象.属性名称=值 这种做法会像对象中添加一个属性

  • 但是obj.arr.push(123)这种结构不会在obj中添加属性，而会在obj.arr数组中添加

二、面向对象

2.1什么是面向对象？

在面向对象之前，都是使用面向过程（一条一条的去做）模式编程，有很多缺点：容易产生命名冲突，不方便代码的重用。

  • 面向对象是一种编程模式，就是以对象的方式编程（构造函数+原型）
  • 实例 ：用面向对象实现把大象装进冰箱

2.2this

  1. 构造函数中的this 指向实例对象
  2. 原型方法中的this 指向实例对象
  3. 事件方法中的this 绑定事件的对象

2.3.1判断数据类型：typeof

  • 数据类型判断 typeof（不是方法，是运算符）

  • 判断出六种数据类型: number string boolean undefined function object (其中null是object)

2.3.2 对象属性判断

  1. in 判断对象中是否存在某个属性（属性实例对象和原型对象都可以）
  2. hasOwnProperty() 判断属性是否在实例对象上
