http协议

http

http是w3c制定的，用来约束客户端浏览器与web服务器进行通讯的数据格式。是基于请求、响应的协议，客户端浏览器给服务器发送一个请求，服务器给客户端一个响应。http协议的数据格式分为两种：请求的数据格式和响应的数据格式。想要研究客户端与浏览器通讯的数据格式，需要用到一个抓包工具：charles

一、★（非常重要）get和post的区别

  1. get提交的数据都在地址栏中，相对来说不安全
  2. get对请求的数据有大小限制
  3. get没有请求体
  4. post请求的参数都在请求体中，相对较安全
  5. post请求对数据的大小没有限制
  6. post必须设置一个特殊请求头
  7. get比post的请求头要少些，性能更高

2.1 一些特殊的请求头

  • post提交有一个特殊的请求头：Content-Type:”application/-w-form-data”
  1. User-Agent：告诉服务器浏览器端的版本
  • 案例：根据User-Agent判断当前浏览器
  • 知识点
    1. getallheaders()方法获取到客户端的请求头，返回的是一个关联数组
    2. strstr() 方法判断某个字符串中是否包含某个字符串,如果包含，就从开始全部截取，如果不包含，就返回false

三、响应的数据格式Response

  • 响应的数据格式
  1. 响应首行
    1.1 状态码（常见的）：
    1.200 代表ok
    2.302 请求重定向
    3.304 请求的资源没有发生改变，从缓存里去找
    4.404 请求的资源没有找到
    5.405 请求的方式不支持
    6.500 服务器内部错误
  2. 响应头
  3. 响应空行
  4. 响应体

三、处理乱码问题

  • 第一种情况 ：客户端的数据发送到服务器
    1. 如果有中文，一般使用post方式提交
  • 第二种方式：服务器的数据响应给客户端
    1. 当前文件采用什么编码
    2. 给客户端一个响应头：Content-Type: text/html;charset = 当前文件的编码
